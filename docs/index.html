Overview

simple_app_api is the top-level unified facade that bundles all application-level services for Eiffel. Instead of managing multiple library dependencies, use a single APP class that provides access to:

  • HTTP client for web requests
  • Alpine.js component generation for reactive UIs
  • All service layer features (JWT, database, email, etc.)
  • All foundation features (Base64, hashing, JSON, UUID, etc.)

This is the top of the API hierarchy: APP inherits SERVICE, which inherits FOUNDATION.

Features

HTTP Client

Make GET, POST, PUT, DELETE requests to web APIs

Alpine.js

Generate reactive UI components with x-data, x-show, x-on

Service Layer

JWT, SMTP, SQL, CORS, rate limiting, templates, WebSocket

Foundation

Base64, hashing, JSON, UUID, CSV, Markdown, validation

Installation

Add simple_app_api to your ECF configuration file:

<library name="simple_app_api"
        location="$SIMPLE_APP_API\simple_app_api.ecf"/>

This single library gives you access to the entire simple_* ecosystem.

Quick Start

local
    app: APP
do
    create app.make

    -- HTTP Client
    client := app.new_web_client
    request := app.new_get_request ("https://api.example.com/users")

    -- Alpine.js Components
    div := app.alpine.div
    div.x_data ("{count: 0}")
    div.x_on_click ("count++")

    -- Service layer (inherited)
    token := app.create_token ("secret", "user", "app", 3600)
    db := app.new_memory_database
    cors := app.new_cors

    -- Foundation layer (inherited)
    encoded := app.base64_encode ("data")
    hash := app.sha256 ("password")
    uuid := app.new_uuid
end

API Reference

Web Client

new_web_client: SIMPLE_WEB_CLIENT
new_get_request (a_url: STRING): SIMPLE_WEB_REQUEST
new_post_request (a_url: STRING): SIMPLE_WEB_REQUEST
web_client: SIMPLE_WEB_CLIENT (singleton)

Alpine.js Components

new_alpine_factory: ALPINE_FACTORY
alpine: ALPINE_FACTORY (singleton)

Inherited from SERVICE

JWT authentication, SMTP email, SQL database, CORS handling, Rate limiting, Template rendering, WebSocket protocol

Inherited from FOUNDATION

Base64 encoding, Hashing (MD5, SHA1, SHA256, HMAC), UUID generation, JSON parsing, CSV processing, Markdown conversion, Validation, Process execution, Random generation

Architecture

The API layers build on each other through inheritance:

APP (Application)

Top layer - HTTP client, Alpine.js components

SERVICE (inherited)

JWT, SMTP, SQL, CORS, rate limiting, templates, WebSocket

FOUNDATION (inherited)

Base64, hashing, JSON, UUID, CSV, Markdown, validation

Bundled Libraries

Library Purpose
simple_webHTTP client and web requests
simple_alpineAlpine.js component generation
simple_service_apiService layer (inherited)
simple_foundation_apiFoundation layer (inherited)

simple_app_api is part of the simple_* library collection.
